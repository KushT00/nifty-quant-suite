How This System Works

The core challenge is data dimensionality. Unlike equity backtesting (2D: time × price), options exist in a 7-dimensional manifold: Spot (S), Strike (K), Time-to-Expiry (τ), Volatility (σ), Risk-Free Rate (r), Option Bias (CE/PE), and Liquidity (OI). This generates a 120× data volume expansion per timestamp compared to spot-only systems.

O(premium) = f(S_t, K, τ, σ, r, Bias, Liquidity) Data Scale: 75 bars/day × 60 strikes × 2 types = 9,000 rows/day/expiry 250 trading days × 18,000 rows = 4.5M rows/year

Architecture decisions: We implemented a lazy-loaded daily cache system (OptionDataLoader) that loads each day's multi-strike CSV once and serves all subsequent bar-level lookups from memory. The Black-Scholes pricer is implemented from scratch (no QuantLib or ta-lib) for zero-dependency portability. The live engines use candle-boundary synchronization — firing evaluations exactly 2 seconds after each 5m close to avoid stale data. On startup, the buying engine replays 5 days of history step-by-step to reconstruct indicator state (warm start), eliminating cold-start blind spots.

Two distinct strategies covering the options spectrum:

1. Option Buying (Directional): Uses a triple-confirmation system (SMA crossover + EMA regime filter + distance check) to identify high-probability trend entries. Buys weekly options closest to ₹200 premium (sweet spot for delta exposure vs. time decay). Protects with a two-phase trailing stop loss on the underlying spot — initial fixed SL of 0.75%, then dynamic trailing once profit exceeds 0.75%. Capital required: ~₹13,000/lot.

2. Option Selling (Theta Harvest): A regime-adaptive system that classifies the market into Ranging/Grey/Trending using Wilder's ADX, then deploys the optimal multi-leg structure: Iron Condors for ranging markets, Ratio Butterflies for trending markets, and Batman Spreads for high-IV ranging. Monitors positions using delta-adjusted premium stops, ITM breach exits, and a weekly circuit breaker. Capital required: ~₹1.5L–2L/lot.
